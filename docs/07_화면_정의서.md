# 07. 화면 정의서 (UI/UX Specification)

본 문서는 GymCRM V2 프론트엔드(`frontend/src/pages/`) 코드베이스에 구현된 실제 화면(Screen)의 레이아웃, 컴포넌트, 연동 데이터(API), 사용자 액션을 명세한 화면 정의서입니다.

---

## 0. 로그인 화면 (Login Page)

시스템에 진입하기 위한 인증 화면입니다.

- **라우트 경로:** `/login`
- **접근 권한:** 전체
- **주요 UI 레이아웃:**
  - **로그인 폼:** 로그인 ID, 비밀번호 입력 필드와 로그인 버튼
  - **안내 문구:** 비밀번호를 잊은 사용자는 현재 센터 관리자에게 문의하고, 관리자가 기존 비밀번호 초기화 절차로 복구한다.
- **주요 액션:**
  - 로그인 ID와 비밀번호를 입력해 운영 콘솔에 진입한다.
  - 비밀번호를 잊은 경우에는 별도 self-service 요청을 만들지 않고 센터 관리자에게 문의한다.

---

## 1. 정산 리포트 화면 (Settlements Page)

매니저와 관리자가 센터의 매출 현황과 트레이너별 정산 금액을 확인하고 확정하는 화면입니다.

- **라우트 경로:** `/settlements`
- **접근 권한:** `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`, `ROLE_MANAGER`, `ROLE_DESK`, `ROLE_TRAINER` (트레이너는 본인 내역만 접근)
- **주요 UI 레이아웃:**
  - **검색 필터 영역:** 연도별, 월별, 주별, 일별 기준일자 선택 및 결제 수단(현금, 카드, 계좌이체 등) 필터링
  - **매출 요약 카드:** 순매출, 총 매출, 총 환불액, 거래 건수 표시 (환불은 붉은색 표기)
  - **미수금 / 후불 후보 카드:** 미수금 총액, 미수금 건수, CRM 적재 후보 수, 기준일/조회 건수 필터와 상세 테이블 표시
  - **트레이너 정산 패널:** 정산 상태(미생성, 초안 생성됨, 확정됨) 표시 및 'preview 완료 수업', 'preview 총액' 조회 탭
- **연동 데이터 (React Query / API):**
  - `useSettlementReportQuery`: 전사 매출 요약 지표 로드
  - `useSettlementReceivablesQuery`: 미수금과 후불 후보 조회
  - `useTrainerSettlementPreviewsQuery`: 조건에 맞는 트레이너별 정산 프리뷰 로드
  - `useCalculateSettlementPreviewsMutation`: 기간에 따른 정산 금액(기본급 + 인센티브 + 수업 수당) 계산 액션
- **주요 액션:**
  - 필터 변경 시 매출 지표 실시간 리렌더링
  - 미수금 카드에서 CRM 적재 후보를 확인하고, 후속 작업이 필요한 건은 CRM 워크스페이스로 이동한다.
  - 개별 트레이너 행 클릭 시 진행한 상세 수업 내역 모달 표출

## 2. 사용자 계정 관리 화면 (User Accounts Page)

센터 내 스태프 계정을 신규 등록하고, 상태 및 권한을 통제하는 화면입니다.

- **라우트 경로:** `/user-accounts`
- **접근 권한:** `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`
- **주요 UI 레이아웃:**
  - **조회 필터 영역:** 이름/로그인 ID 검색창, 역할 필터, 상태(ACTIVE/INACTIVE) 필터
  - **데이터 테이블:** 로그인 ID, 이름, 역할, 상태, 비밀번호 상태(임시/정상), 최근 로그인, 접속 제한 시점 표시
  - **작업 버튼 그룹:** 각 행마다 [역할 변경], [접속 취소], [비활성화/활성화] 버튼 노출
  - **모달 (계정 생성):** ID, 이름, 역할 선택, 임시 비밀번호 입력 폼
  - **안내 문구:** 비밀번호를 잊은 스태프는 현재 센터 관리자에게 문의하고, 관리자는 기존 비밀번호 초기화 절차로 복구한다.
- **연동 데이터 (React Query / API):**
  - `useUserAccountsQuery`: 페이지네이션 기반 사용자 계정 목록 로드
  - `useAccountOpsMutations`: 계정 생성(`createAccount`), 토큰 만료(`revokeAccess`), 권한 변경(`changeRole`), 상태 변경(`changeStatus`) API 호출
- **주요 액션:**
  - **신규 계정 생성:** 비밀번호 정책(8자 이상, 문자/숫자/특수문자 조합) 프론트엔드 폼 검증 로직 적용
  - **접속 권한 취소:** 특정 사용자의 세션 토큰을 즉각 무효화(Access Revoked)하여 강제 로그아웃 처리

## 3. 회원권 업무 화면 (Memberships Page)

선택된 특정 회원의 회원권 구매, 홀딩(일시정지), 재개, 환불 등 '생애주기 콘솔' 역할을 하는 핵심 화면입니다.

- **라우트 경로:** `/memberships`
- **접근 권한:** `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`, `ROLE_MANAGER`, `ROLE_DESK`
- **주요 UI 레이아웃:**
  - **컨텍스트 패널:** 선택된 회원의 이름, 회원 ID, 총 회원권 건수, 세션 결제 건수 표시
  - **활성 회원권 테이블:** 선택된 회원의 보유 회원권 리스트 (ID, 상품, 상태 배지, 유효기간, 잔여 횟수)
  - **결제 이력 카드:** 방금 실행한 작업(구매/환불 등)의 결제 결과를 우측 패널에 즉시 표출
  - **작업 모달 4종:** [신규 등록], [홀딩], [재개], [환불] 
- **연동 데이터 (React Query / API):**
  - `useSelectedMemberMembershipsQuery`: 특정 회원의 회원권 상태 및 액션(구매/홀딩/환불) Mutation 관리
  - `useProductsQuery`: 등록 가능한 상품 리스트 로드
  - `useTrainerOptionsQuery`: PT 상품 구매 시 할당할 트레이너 목록 로드
- **주요 액션 및 비즈니스 로직:**
  - **회원권 홀딩:** 시작/종료일 선택 시, 상품에 부여된 '잔여 일수' 및 '최대 홀딩 횟수' 검증. 제한 초과 시 붉은색 경고 출력 및 권한 기반 **[강제 홀딩 처리]** 플로우 제공.
  - **회원권 환불:** 환불 기준일 및 수단 선택. 결제 금액에서 이용분 공제 및 위약금(결제액의 10%)을 프론트엔드에서 계산하여 최종 환불 금액 실시간 프리뷰.
  - **신규 등록 (구매):** 결제 수단(현금, 카드 등)과 할인율 적용된 실 결제 금액(Paid Amount) 입력 기능 제공.

## 3.1 상품 관리 화면 (Products Page)

센터의 상품 카탈로그를 등록, 수정, 비활성화하고 PT 상품에는 담당 트레이너를 연결하는 화면입니다.

- **라우트 경로:** `/products`
- **접근 권한:** `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`, `ROLE_MANAGER`, `ROLE_DESK` (조회), `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`, `ROLE_MANAGER` (수정)
- **주요 UI 레이아웃:**
  - **요약 카드:** 전체 상품 수, 활성 상품 수, 평균 판매가, 시스템 권한 상태를 표시한다.
  - **필터 영역:** 상품 분류와 판매 상태를 필터링한다.
  - **상품 테이블:** 상품명, 분류, 가격, 담당 트레이너, 프로모션, 상태, 액션을 표시한다.
  - **편집 모달:** 상품명, 분류, 유형, 금액, 기간/횟수, 프로모션 기간/할인값, 홀딩 정책, 담당 트레이너(PT 전용), 내부 설명을 편집한다.
- **연동 데이터 (React Query / API):**
  - `useProductsQuery`: 상품 목록 로드
  - `useTrainerOptionsQuery`: PT 상품에 연결할 트레이너 선택지 로드
  - `useProductPrototypeState`: 상품 생성/수정/상태 전환 액션 관리
- **주요 액션:**
  - PT 상품은 담당 트레이너를 반드시 지정해야 한다.
  - 비-PT 상품은 담당 트레이너 필드를 비워 두며, 목록에서는 해당 없음으로 표시한다.
  - 상품 수정 시 현재 PT 연결 상태를 유지하거나 트레이너를 변경할 수 있다.
  - 프로모션이 설정된 상품은 목록에서 할인율/금액과 기간을 함께 표시하고, 편집 모달에서 기간과 할인값을 함께 조정한다.

## 3.2 라커 관리 화면 (Lockers Page)

센터의 라커 재고, 배정 현황, 구역/등급별 월 이용료를 확인하고 신규 라커를 일괄 등록하는 화면입니다.

- **라우트 경로:** `/lockers`
- **접근 권한:** `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`, `ROLE_MANAGER`, `ROLE_DESK` (조회/배정), `ROLE_SUPER_ADMIN`, `ROLE_ADMIN` (라커 등록)
- **주요 UI 레이아웃:**
  - **요약 카드:** 전체 라커 수, 사용 가능 라커 수, 현재 배정 수, 시스템 권한 상태를 표시한다.
  - **라커 목록 테이블:** 라커 코드, 구역, 등급, 월 이용료, 상태, 메모를 표시한다.
  - **현재 배정 현황 테이블:** 라커 코드, 회원, 사용 기간, 반납 액션을 표시한다.
  - **배정 모달:** 대상 라커, 회원, 시작일, 종료일, 운영 메모를 입력해 신규 배정을 등록한다.
  - **일괄 등록 모달:** 구역, 번호, 자동 코드 프리뷰, 등급, 상태, 메모, 월 이용료를 행 단위로 입력해 다건 등록한다.
- **연동 데이터 (React Query / API):**
  - `useLockerQueries`: 라커 목록과 현재 배정 현황 로드
  - `useLockerPrototypeState`: 라커 배정, 반납, 일괄 등록 액션 관리
  - `useMembersQuery`: 배정 대상 회원 선택지 로드
- **주요 액션:**
  - 라커 등록 시 구역 + 번호로 코드가 자동 생성되고, 월 이용료는 숫자 입력으로 저장된다.
  - 라커 배정은 선택한 라커와 회원, 시작/종료일을 함께 제출해야 한다.
  - 배정 중인 라커는 목록과 배정 현황에서 함께 확인하고 반납할 수 있다.
  - 권한이 없는 사용자는 조회만 가능하며 등록/배정 작업은 잠긴다.

## 4. 회원 관리 화면 (Member List Page)

전체 회원을 검색하고 상태를 조회하며, 회원권 업무를 수행하기 위해 특정 회원을 선택(Context Select)하는 화면입니다.

- **라우트 경로:** `/members`
- **접근 권한:** `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`, `ROLE_MANAGER`, `ROLE_DESK`, `ROLE_TRAINER`
- **주요 UI 레이아웃:**
  - **검색 패널:** 이름, 연락처 기반 검색창
  - **회원 리스트 테이블:** 회원 아바타(썸네일), 회원 이름, 연락처, 담당 트레이너, 회원권 운용 상태 표시
  - **사진 업로드:** 회원 행 또는 상세에서 사진 업로드 버튼 클릭 시 파일 선택 다이얼로그 → `POST /api/v1/members/{memberId}/photo`(multipart/form-data) 호출 → 저장 성공 시 아바타 썸네일 즉시 갱신
  - **상태 배지:** `정상(Success 색상)`, `홀딩중(Warning 색상)`, `만료임박`, `만료` 등 직관적인 태그 컴포넌트 렌더링
- **주요 액션:**
  - 회원 목록에서 특정 회원을 클릭하여 **SelectedMemberContext** 전역 상태 업데이트 후 `[회원권 업무]` 등 다른 화면으로 라우팅.

## 5. 시스템 설정 화면 (Center Settings Page)

센터의 프로필 정보를 설정하는 기본 화면입니다.

- **라우트 경로:** `/settings`
- **접근 권한:** `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`
- **주요 UI 레이아웃:**
  - **센터 프로필 카드:** 읽기 모드(Descriptions 컴포넌트)와 수정 모드(Form 컴포넌트) 상호 전환 기능
- **연동 데이터 (React Query / API):**
  - `useCenterProfileQuery`: 센터의 이름, 전화번호, 주소 로드
  - `useUpdateCenterProfileMutation`: 센터 프로필 업데이트 액션
- **주요 액션:**
  - [수정] 버튼 클릭 시 입력 폼으로 전환되며, [저장] 시 즉각적인 성공 메시지(Message 팝업) 피드백 제공.

## 6. CRM 운영 화면 (CRM Page)

관리자가 CRM 템플릿 거버넌스와 메시지 적재/이력 상태를 한 화면에서 운영하는 화면입니다.

- **라우트 경로:** `/crm`
- **접근 권한:** `ROLE_SUPER_ADMIN`, `ROLE_ADMIN`, `ROLE_MANAGER`, `ROLE_DESK`
- **주요 UI 레이아웃:**
  - **템플릿 거버넌스 카드:** 템플릿 심사 상태, 운영 상태, 발송 가능 여부, 활성 여부를 조회하고 채널/활성 필터를 적용한다.
  - **큐 자동화 제어 카드:** 만료 안내 D-Day 입력과 메시지 큐 실행 버튼을 제공한다.
  - **장기 미방문 캠페인 카드:** 선택된 sendable 템플릿을 기준으로 장기 미방문 일수와 예약 발송 시각을 입력해 캠페인을 적재한다.
  - **발송 로그 및 이력 카드:** 발송 상태, 시도 횟수, 발송 경로(`PRIMARY`, `SMS_FALLBACK`)와 실패 힌트를 표시한다.
- **연동 데이터 (React Query / API):**
  - `useCrmTemplatesQuery`: CRM 템플릿 목록과 거버넌스 상태 로드
  - `useCrmHistoryQuery`: CRM 발송 이력 로드
  - `useCrmPrototypeState`: 만료 안내 적재, 큐 실행, 장기 미방문 캠페인 적재 액션 관리
- **주요 액션:**
  - 템플릿 목록에서 sendable 템플릿을 선택하고 캠페인 적재에 재사용한다.
  - 장기 미방문 캠페인은 예약 발송 시각을 입력하면 동일한 큐/이력 경로에서 대기 상태로 보관된다.
  - 실패한 발송은 이력에서 최종 실패와 폴백 경로를 함께 확인한다.

## 10. 회원 QR 화면 (Member QR Page)

회원이 휴대폰에서 서명된 QR 링크를 열었을 때 출입용 QR을 보여 주고, 만료 전에 다시 갱신할 수 있는 모바일 전용 화면입니다.

- **라우트 경로:** `/member-qr`
- **접근 권한:** 공개 링크 토큰 필요(로그인 불필요)
- **주요 UI 레이아웃:**
  - **헤더 영역:** 회원 이름이 포함된 QR 제목, 짧은 안내 문구
  - **QR 카드:** 현재 QR 코드, 상태 배지, 남은 시간, 링크 만료 시각 표시
  - **액션 버튼:** QR 새로고침 버튼과 재시도 안내
  - **오류 상태:** 링크 토큰이 없거나 만료된 경우 명확한 재시도 메시지 표시
- **연동 데이터 (React Query / API):**
  - `POST /api/v1/access/qr/member-session`: bootstrapToken을 검증하고 현재 QR 세션을 다시 발급
  - `POST /api/v1/access/qr/member-link`: 운영자가 회원 QR 링크를 재발급하는 보조 API
- **주요 액션:**
  - 링크 토큰이 있으면 페이지 진입 시 즉시 현재 QR을 받아온다.
  - 남은 시간과 만료 시각을 함께 보여 주어 회원이 QR 상태를 이해할 수 있게 한다.
  - 회원은 명시적으로 발급된 `memberQrPath`를 즐겨찾기, 메신저 대화, 홈 화면 바로가기 등으로 저장해 다시 열 수 있다.
  - 세션이 짧게 만료되거나 토큰이 무효한 경우 빈 화면 대신 재시도 가능한 오류 상태를 보여 준다.
  - 회원 등록/수정 화면은 필요 시 QR 접근 링크를 즉시 열 수 있게 하고, 회원 상세 화면은 QR 링크를 자동 노출하지 않는다.
