# Dashboard Theme Refactor (Light/Dark) Brainstorm

## What We're Building
기존 GymCRM UI를 레퍼런스 이미지의 구조/톤에 맞춰 대시보드 중심으로 시각 리팩터링한다. 1차 목표는 대시보드 화면에 한정해 정보 밀도, 카드 위계, 사이드바/탑바 인상을 개선하는 것이다. 이후 2차로 동일한 테마 체계(토큰/컴포넌트 상태)를 회원/상품/예약 등 전체 탭에 일괄 확장한다.

동시에 라이트/다크 모드를 지원하며, 기본 동작은 시스템 설정(`prefers-color-scheme`)을 따른다. 사용자가 수동으로 테마를 전환하면 해당 선택을 로컬 저장해 이후 방문 시 우선 적용한다.

## Why This Approach
단계적 적용(대시보드 우선 → 전체 탭 확장)은 리스크를 낮추고 빠른 검증이 가능하다. 현재 화면에서 가장 시각적 영향이 큰 영역은 대시보드이며, 이 영역에서 디자인 토큰/컴포넌트 스타일 패턴을 먼저 확정하면 이후 탭 확장 시 중복 수정과 회귀를 줄일 수 있다.

테마 정책은 “시스템 설정 우선 + 사용자 오버라이드 저장”이 UX 기대와 일치한다. 초기 진입에서 사용자의 OS 환경을 존중하고, 명시적 선호를 저장해 제어권도 제공한다.

## Key Decisions
- 대시보드 우선 적용: 1차 범위를 대시보드로 제한해 빠르게 품질 검증 후 전체 탭 확장.
- 최종 목표는 전 탭 일괄 적용: 대시보드 전용 스킨이 아니라 공통 토큰 체계를 전제로 설계.
- 테마 기본값: 시스템 설정을 따름.
- 사용자 수동 전환: localStorage에 저장해 이후 재방문 시 반영.
- 전환 UI 위치: TopBar 우측 토글 스위치.
- 전환 UX: 즉시 반영, 애니메이션 최소(과한 transition 지양).

## Constraints / Guardrails
- 1차에서 기능 로직(API/state flow) 변경 없이 시각/레이아웃/테마 계층에 집중.
- 대시보드 외 탭은 1차에서 기능 회귀가 없어야 함.
- 접근성 기준 유지: 대비, 포커스 가시성, 토글 조작성(키보드 포함).

## Success Criteria (Phase 1)
- 대시보드에서 TopBar 우측 토글 조작 시 라이트/다크 테마가 즉시 반영된다.
- 사용자 수동 전환 시 선택값이 localStorage에 저장되고, 재방문 시 동일 테마가 복원된다.
- 저장된 사용자 선택이 없을 때는 시스템 설정(`prefers-color-scheme`)을 기본 테마로 적용한다.
- 테마 전환 시 과한 시각 효과 없이 최소 애니메이션으로 전환되며, 콘텐츠 가독성이 유지된다.
- 대시보드 외 탭에서 기능/상태/API 동작 회귀가 발생하지 않는다.
