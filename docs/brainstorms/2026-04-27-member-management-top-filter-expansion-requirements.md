# Member Management Top Filter Expansion Requirements

## Problem Frame
회원관리 화면은 이미 이름, 연락처, 회원 상태, 회원권 상태, `만료 기간` 필터를 제공하지만, 운영자가 실제 업무 맥락으로 회원을 빠르게 좁히는 축이 아직 부족하다. 이번 `Upcoming Roadmap`의 남은 페이지 범위는 회원관리 화면의 상단 필터 확장으로 고정되어 있으므로, 예약/업무 워크스페이스까지 번지지 않게 범위를 분리한다.

이 작업의 핵심은 운영자가 회원을 `담당 트레이너`, `상품`으로 더 정확하게 찾을 수 있게 만드는 것이다. 기존 `만료 기간` 필터는 그대로 유지하고, 이번 확장은 그 위에 업무용 필터 축을 추가하는 방향으로 잡는다.

## Requirements

**필터 확장**
- R1. 회원관리 상단 필터에 `회원권에 연결된 담당 트레이너` 필터를 추가한다.
- R2. 회원관리 상단 필터에 `회원권 상품` 필터를 추가한다.
- R3. 기존 `만료 기간` 필터의 의미와 동작은 그대로 유지한다.
- R4. 새 필터는 기존 회원 목록 필터와 함께 조합되어 동작해야 한다.
- R5. `담당 트레이너`와 `회원권 상품` 필터는 회원의 회원권 정보를 기준으로 판정하며, 같은 회원에 여러 회원권이 있으면 회원 목록 요약과 일관된 범위를 따라야 한다.

**검색 행위**
- R6. 필터 변경 후 조회 결과는 회원관리 목록의 조회 동작에 반영되어야 한다.
- R7. 필터 초기화 시 기존 필터와 새로 추가된 필터가 모두 함께 초기화되어야 한다.
- R8. 필터 조합이 바뀌어도 회원 목록의 페이징 동작은 유지되어야 하며, 결과 탐색이 끊기지 않아야 한다.

**표현 규칙**
- R9. `담당 트레이너`와 `회원권 상품`은 운영자가 의미를 오해하지 않도록 회원 관리 화면의 기존 맥락과 같은 용어로 노출되어야 한다.
- R10. 기존 `만료 기간` 필터는 화면상에서도 만료 예정 조회라는 현재 해석이 유지되도록 표현되어야 한다.

## Success Criteria
- 회원관리 화면에서 운영자가 회원을 트레이너와 상품 기준으로 더 좁힐 수 있다.
- 회원권에 연결된 담당 트레이너와 회원권 상품을 기준으로 필터가 동작한다.
- 기존 이름, 연락처, 회원 상태, 회원권 상태 필터와 새 필터가 충돌하지 않고 함께 동작한다.
- 기존 `만료 기간` 필터의 동작이 유지된다.
- 초기화 시 필터 상태가 모두 원복된다.

## Scope Boundaries
- 예약 관리 워크스페이스 개편은 이번 범위에 포함하지 않는다.
- 회원 상세 모달의 정보 확장이나 액션 변경은 이번 범위에 포함하지 않는다.
- 회원 목록 컬럼 추가/개편은 이번 범위에 포함하지 않는다.
- 회원/권한 모델 변경은 이 요구사항의 목표가 아니다.
- 기존 `만료 기간` 필터의 의미 변경은 이번 범위에 포함하지 않는다.

## Key Decisions
- 기존 `만료 기간` 필터는 유지한다. `기간` 의미를 바꾸지 않고, 새 필터 축을 추가하는 쪽이 기존 사용성에 안전하다.
- 이번 확장은 `회원관리 상단 필터`에만 적용한다. 다른 화면의 공통 검색 경험으로 일반화하는 것은 이번 단계에서 하지 않는다.
- 새 필터는 기존 필터 체계 위에 얹는 방식으로 간다. 기존 검색을 대체하지 않고, 더 좁은 조회 축을 추가하는 방향이 가장 적은 carrying cost를 만든다.

## Dependencies / Assumptions
- `담당 트레이너`와 `상품`은 회원 목록에서 필터 가능한 의미 있는 축으로 식별 가능해야 한다.
- 회원 목록 조회의 현재 필터/요약 규칙은 유지된다고 가정한다.
- 날짜 해석은 현재 프로젝트의 로컬 비즈니스 날짜 규칙을 따른다고 가정한다.
