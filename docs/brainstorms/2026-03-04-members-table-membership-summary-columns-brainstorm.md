# Members Table Membership Summary Columns Brainstorm

## What We're Building
회원관리 탭의 회원 목록 테이블을 운영 관점에서 확장한다. 기존 컬럼(ID, 이름, 연락처, 상태, 가입일, 액션)만으로는 회원권 상태를 빠르게 판단하기 어려워, 회원권 요약 정보를 목록에서 즉시 확인할 수 있게 한다.

추가 대상은 3가지다: 운영용 단순 상태(정상/만료임박/만료/없음), 대표 회원권 만료일, 남은 PT 횟수. 남은 PT 횟수는 기간형 회원권과 분리된 별도 열로 표시하며, 값이 없거나 표시 대상이 아니면 `-`로 노출한다.

## Why This Approach
운영자는 목록에서 “누가 곧 만료되는지, PT 소진 상태가 어떤지”를 한 번에 판단해야 한다. 회원 상세/회원권 탭으로 이동해 확인하는 방식은 반복 비용이 크다.

데이터 제공 방식은 프론트 조합이 아니라 회원 목록 API 확장을 선택한다. 이 방식이 성능(N+1 회피), 페이징 일관성, 정렬/필터 확장성 측면에서 가장 안정적이며, 프론트 복잡도도 낮다.

## Approaches Considered
### Approach A (Selected): 회원 목록 API 확장
- 회원 목록 응답에 운영 요약 필드(운영상태/대표만료일/PT잔여횟수)를 포함.
- 장점: 성능과 일관성이 좋고 화면 렌더링이 단순함.
- 단점: 백엔드 DTO/쿼리 수정 필요.

### Approach B: 프론트엔드 합성
- 기존 회원 목록 + 회원별 추가 조회로 데이터 조합.
- 장점: 초기 백엔드 변경 최소.
- 단점: N+1 호출, 로딩 복잡도, 목록 정렬/페이징 정합성 저하.

## Key Decisions
- 대표 회원권 선정 기준: `ACTIVE` 우선, 그 안에서 `만료일이 가장 빠른 1건`.
- 상태 표시는 원본 상태코드가 아닌 운영용 단순 상태로 가공.
- 운영용 상태 값: `정상`, `만료임박`, `만료`, `없음`.
- 만료임박 기준: 만료일 `7일 전`.
- PT 잔여는 별도 열로 표시.
- PT 잔여 표기 규칙: 값이 있으면 숫자, 없거나 비대상이면 `-`.
- 데이터 소스: 권장안 채택, 회원 목록 API 확장으로 제공.

## Rule Clarifications
- 대표 회원권 선정 시 `ACTIVE`가 없으면 대표 회원권을 `없음`으로 처리한다.
- `ACTIVE` 회원권이 여러 개인 경우:
  - 1차: 만료일 오름차순(가장 빠른 만료일 우선)
  - 2차: 동일 만료일이면 생성시각/ID 오름차순 등 안정적인 tie-break를 사용한다.
- 대표 회원권 만료일 표기:
  - 유효한 만료일이 있으면 날짜 표시
  - 없으면 `-` 표시
- PT 잔여 횟수는 “회원이 현재 사용 가능한 PT 횟수 총합”을 기준으로 한다.
  - 0 이하이거나 사용 가능한 PT가 없으면 `-` 표시
- 만료임박 7일 계산은 센터 운영 기준 날짜(로컬 타임존 기준 일 단위)로 판정한다.

## Status Mapping (Operational)
- `정상`: 대표 회원권이 `ACTIVE`이고
  - 만료일이 없는 경우(횟수형/무기한 성격) 또는
  - 만료일까지 8일 이상 남은 경우
- `만료임박`: 대표 회원권이 `ACTIVE`이고 만료일이 있으며 만료일까지 0~7일 남은 경우
- `만료`: 대표 회원권이 만료일 기준으로 이미 지난 경우
- `없음`: 대표 회원권 자체가 없거나, 상태 판단에 필요한 최소 데이터가 없는 경우

## Data Contract Direction
- 회원 목록 응답에 운영 요약 필드를 포함한다.
  - `membershipOperationalStatus` (정상/만료임박/만료/없음)
  - `membershipExpiryDate` (YYYY-MM-DD | null)
  - `remainingPtCount` (number | null)
- 프론트는 요약 필드를 그대로 렌더링하고, 표현 규칙(`null`/0 → `-`)만 담당한다.

## Constraints / Guardrails
- 회원 목록 조회 성능은 기존 수준을 유지하거나 개선해야 한다.
- 대표 회원권 선정/상태 가공 규칙은 목록 전반에 일관되게 적용한다.
- 테이블 확장 시 모바일/좁은 뷰포트 가독성 저하를 방지해야 한다.
- 기존 회원 선택/액션(보기/수정/회원권업무/예약관리) 동작 회귀가 없어야 한다.
- 목록 페이징/정렬/필터 결과와 요약 필드가 불일치하지 않아야 한다.

## Success Criteria
- 회원 목록에서 운영용 상태/대표 만료일/PT 잔여를 별도 열로 즉시 확인할 수 있다.
- 운영용 상태가 합의된 규칙(정상/만료임박/만료/없음, 7일 기준)에 맞게 표시된다.
- PT 잔여 표기는 숫자 또는 `-` 규칙을 일관되게 따른다.
- 목록 로딩/스크롤/필터 사용감이 유의미하게 저하되지 않는다.
- 기존 회원관리 핵심 액션에 기능 회귀가 없다.
- 대표 회원권/만료임박 판정이 테스트 데이터 케이스(복수 ACTIVE, 만료일 없음, 경계일 D-7/D-8)에서 일관되게 동작한다.
