# GymCRM V2 내부 운영 사용자 매뉴얼

## Problem Frame
GymCRM V2를 실제로 운영하는 관리자와 직원이 여러 기능을 빠르게 익히고, 업무 중 다시 찾아볼 수 있는 한 문서가 필요하다. 기존의 수동 점검 체크리스트나 상급자 보고용 요약본만으로는 신규 온보딩과 실무 레퍼런스를 동시에 충족하기 어렵다. 결과물은 PDF로 공유할 수 있어야 하고, 같은 내용을 짧게 압축한 위키용 요약본도 함께 쓸 수 있어야 한다.

## Requirements

**[Deliverables & Format]**
- R1. 문서는 저장소 내 단일 마크다운 원문을 기준으로 작성되어야 한다.
- R2. 원문은 PDF로 변환하기 쉬운 구조여야 한다.
- R3. 같은 내용을 바탕으로 사내 위키에 올릴 수 있는 짧은 요약본도 함께 준비되어야 한다.
- R4. 본문은 인쇄/배포용, 요약본은 빠른 참조용으로 역할이 분리되어야 한다.

**[Audience & Usage]**
- R5. 대상은 내부 관리자와 직원이며, 신규 온보딩과 실무 레퍼런스 용도를 모두 만족해야 한다.
- R6. 각 기능은 "무엇을 하는 화면인지", "누가 쓰는지", "언제 쓰는지"를 먼저 설명해야 한다.
- R7. 클릭 순서 중심의 장문 튜토리얼보다, 업무 목적과 핵심 사용법 위주의 안내로 작성해야 한다.

**[Coverage]**
- R8. 관리자 화면의 주요 기능을 폭넓게 다뤄야 한다.
- R9. 최소한 대시보드, 회원 관리, 회원권 업무, 예약 관리, 상품 관리, 트레이너 관리, GX 스케줄, 출입 관리, 라커 관리, CRM 메시지, 정산 리포트, 감사 로그, 시스템 설정, 사용자 계정 관리, 내 스케줄, 내 계정 관련 흐름을 포함해야 한다.
- R10. 역할별로 접근 가능한 기능 차이를 문서 안에서 구분해 안내해야 한다.
- R11. 각 기능 설명에는 목적, 주요 입력/출력, 주의할 점, 자주 쓰는 작업을 포함해야 한다.

**[Structure]**
- R12. 문서는 시작 부분에 시스템 개요와 읽는 방법을 포함해야 한다.
- R13. 기능 본문은 모듈별 장으로 구성하고, 필요하면 역할별 섹션으로 보완해야 한다.
- R14. 전체 기능을 나열한 뒤, 실제 업무 흐름 기준의 빠른 시작 섹션을 별도로 제공해야 한다.
- R15. 결론부에는 자주 묻는 질문 또는 문제 해결 팁을 짧게 포함해야 한다.
- R16. 중요한 화면에는 이미지 자리표시자 또는 캡처 삽입 위치를 남겨야 한다.

## Success Criteria
- 신규 직원이 매뉴얼만 보고 주요 업무 화면의 역할과 사용 목적을 이해할 수 있다.
- 현장 직원이 업무 중 필요한 화면을 빠르게 찾을 수 있다.
- PDF 변환 시 레이아웃이 깨지지 않고 읽기 쉬운 형태로 배포된다.
- 위키 요약본은 긴 본문을 읽지 않아도 핵심 기능을 파악할 수 있다.

## Scope Boundaries
- 최종회원용 고객 안내서는 제외한다.
- 세세한 버튼 클릭 단계별 캡처 가이드는 제외한다.
- 개발자용 API 문서, 배포 문서, 장애 대응 런북은 제외한다.
- 별도 웹 포털이나 인터랙티브 도움말 시스템 구축은 제외한다.

## Key Decisions
- 문서 형태는 `PDF용 원문 + 위키용 요약본`의 혼합형으로 간다.
- 본문 구성은 기능별 안내를 기본으로 하고, 초반에 빠른 시작 섹션을 둔다.
- 상세 절차보다 업무 목적, 권한, 핵심 화면, 주의사항을 우선한다.
- 기존의 보고용 Quick Reference 방향은 이번 요구와 분리하고, 내부 운영 매뉴얼 기준으로 새로 작성한다.

## Proposed Structure

**0. Front Matter**
- 표지
- 개정 이력
- 읽는 방법

**1. 시스템 개요**
- GymCRM V2가 무엇을 하는 시스템인지
- 역할별 사용자 구분
- 문서에서 사용하는 용어

**2. 공통 사용 규칙**
- 로그인과 세션
- 권한과 접근 제한
- 공통 화면 구성
- 자주 보는 상태값과 알림
- 화면에서 자주 보는 버튼과 필터 용어

**3. 빠른 시작**
- 신규 직원이 가장 먼저 보는 5가지 업무
- 회원 찾기
- 회원권 확인
- 예약 확인
- 상품 확인
- 정산/계정 확인

**4. 회원 관리**
- 회원 목록
- 회원 상세
- 회원 등록/수정
- 회원 상태 관리
- 회원 검색과 필터
- 회원 상세에서 확인하는 핵심 정보
- 신규 회원 등록 시 확인 항목
- 수정/비활성화 시 주의사항

**5. 회원권 업무**
- 회원권 조회
- 구매/연장/홀딩/해제
- 환불
- 만료 및 상태 해석
- 회원권 상태별 의미
- 회원권을 붙일 때와 끊을 때 확인할 것
- 홀딩과 해제의 운영 기준
- 환불 전에 확인할 항목

**6. 예약 관리**
- 예약 목록
- 예약 생성/변경/취소
- 예약 대상과 회원권 연계
- 예약 가능한 조건
- 예약 상태별 처리 방식
- 예약 변경/취소 시 영향
- 현장 운영에서 자주 보는 예외

**7. 상품 관리**
- 상품 목록
- 상품 등록/수정
- 상품 유형별 확인 포인트
- 상품 유형별 차이
- 금액/횟수/기간 기준 확인
- 노출/비노출 판단 기준

**8. 트레이너 관리**
- 트레이너 목록
- 트레이너 등록/수정
- 담당 범위와 상태 확인
- 트레이너 계정과 인적 정보 구분
- 담당 회원/수업/일정 확인
- 활성/비활성 상태 의미

**9. GX 스케줄**
- 스케줄 목록
- 등록/변경/운영 확인
- 수업명/트레이너/시간/정원 확인
- 주간/월간 운영에서 보는 포인트
- 변경 시 영향 범위

**10. 출입 관리**
- 출입 조회
- 출입 상태 확인
- 운영 시 주의사항
- 출입 기록을 보는 목적
- 출입 이상 징후 확인
- 권한별 확인 가능 범위

**11. 라커 관리**
- 라커 목록
- 배정/해제
- 상태 확인
- 배정 가능 상태와 점유 상태
- 만료/해제 시 처리
- 현장 확인 포인트

**12. CRM 메시지**
- 메시지 목록
- 발송/이력 확인
- 운영 시 주의사항
- 메시지 발송 목적
- 대상 선택 시 주의사항
- 발송 후 이력 확인 방법

**13. 정산 리포트**
- 정산 리포트 개요
- 역할별 열람 범위
- 자주 보는 지표
- 월별/기간별 확인 기준
- 관리자와 트레이너의 보는 관점 차이
- 숫자를 해석할 때 주의할 점

**14. 감사 로그**
- 로그 확인 목적
- 추적할 때 보는 항목
- 누가/언제/무엇을 했는지 확인
- 문제 발생 시 검색 순서
- 운영 이력과 감사 이력의 차이

**15. 시스템 설정**
- 센터 설정
- 운영 파라미터 확인
- 설정 변경 전 확인할 항목
- 운영에 영향이 큰 설정
- 변경 후 점검해야 할 화면

**16. 사용자 계정 관리**
- 계정 목록
- 계정 생성/변경
- 비밀번호 및 권한 관련 주의사항
- 계정 생성과 역할 부여
- 권한 변경 시 영향
- 비밀번호 초기화/변경 시 확인 사항

**17. 내 스케줄 / 내 계정**
- 트레이너 전용 내 스케줄
- 내 계정 정보
- 비밀번호 변경이 필요한 경우
- 본인 일정 확인 흐름
- 개인정보 확인 범위
- 계정 보안 유지 방법

**18. 문제 해결과 FAQ**
- 자주 발생하는 권한 문제
- 조회가 안 될 때 확인할 것
- 상태값이 헷갈릴 때 보는 기준
- 로그인 관련 문제
- 데이터가 안 보일 때 확인할 것
- 권한이 부족할 때 대응 방법

**19. 부록**
- 용어집
- 화면 캡처 목록
- 위키 요약본 링크
- 권한별 기능 빠른 표
- 자주 쓰는 메뉴 경로 표
- 문서 개정 이력

## Dependencies / Assumptions
- 화면 구성과 기능 명칭은 현재 관리자 UI 라우트와 실제 페이지 기준으로 정리한다.
- 스크린샷은 필요시 나중에 추가할 수 있는 자리표시자 방식으로 잡는다.
