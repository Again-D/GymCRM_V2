# GX Recurring Schedule Registration

## What We're Building
GX 수업 스케줄 등록은 개별 회차를 하나씩 만드는 방식이 아니라, 반복되는 고정 수업을 중심으로 운영할 수 있는 등록 모델을 목표로 한다. 운영자는 수업명, 담당 강사, 요일, 시작/종료 시간, 정원, 적용 시작일을 기준으로 반복 규칙을 등록하고, 시스템은 이 규칙을 바탕으로 앞으로 4주치 실제 예약 가능한 수업 슬롯을 유지한다.

회원 예약과 정원 관리는 항상 생성된 실제 슬롯 기준으로 처리한다. 반복 규칙은 운영 편의를 위한 상위 개념이고, 예약/출석/취소/정원 차감은 개별 슬롯에 귀속되는 구조를 유지한다. 즉, 운영자는 반복 규칙을 관리하고 회원은 생성된 회차를 예약하는 구조로 역할을 분리한다.

## Why This Approach
GX는 대부분 매주 반복되는 고정 수업이 주를 이룰 것으로 보이므로, 날짜별 수동 등록을 기본 UX로 두는 것은 운영 부담이 크다. 반대로 반복 규칙만 저장하고 조회 시 실시간으로 슬롯을 계산하는 방식은 예외 처리, 예약 상태 관리, 정원 관리가 불필요하게 복잡해진다.

반복 규칙 기반으로 실제 슬롯을 미리 생성하는 방식은 운영성과 단순성을 동시에 확보한다. 기존 예약 구조도 실제 슬롯 단위로 관리되고 있어, 이 방향이 현재 코드베이스의 예약 모델과도 가장 자연스럽게 맞는다. 특히 반복 규칙과 개별 슬롯을 분리하면, 운영 화면은 단순하게 유지하면서도 특정 날짜 휴강이나 강사 변경 같은 예외를 안전하게 처리할 수 있다.

## Key Decisions
- 반복 수업 중심 등록: GX는 대부분 고정 반복 수업이므로, 등록의 기본 단위는 개별 날짜 슬롯이 아니라 반복 규칙으로 둔다.
- 실제 예약은 생성 슬롯 기준: 회원 예약과 정원 관리는 항상 실제 생성된 슬롯에 연결해 현재 예약 모델의 단순성을 유지한다.
- 생성 범위는 4주: 운영상 충분한 가시성을 주면서도 과도한 미래 데이터 생성을 피하기 위해 앞으로 4주치 슬롯을 롤링 윈도우로 유지한다.
- 예외는 수정형으로 시작: 특정 날짜에 대해 휴강, 강사 변경, 시간 변경, 정원 변경을 지원하되, 회차를 완전히 독립 편집하는 모델은 1차 범위에서 제외한다.
- 예외는 기본 규칙에 종속된다: 특정 날짜 예외는 반복 규칙을 깨지 않고 해당 회차에만 적용되는 수정으로 취급한다.
- 권한은 운영 설계와 현장 대응으로 분리한다: 반복 규칙의 생성/수정/종료는 센터 매니저 권한으로 두고, 트레이너는 본인에게 배정된 회차에 한해 제한적인 예외 처리만 허용한다.
- 독립 편집형은 후순위: 특정 회차를 별도 수업으로 분리하는 복잡한 모델은 실제 운영 필요가 확인될 때 2차 기능으로 검토한다.

## Resolved Questions
- GX 수업은 대부분 매주 반복되는 고정 수업이 주를 이룬다.
- 예외 처리는 특정 날짜 휴강/수정 수준으로 시작하는 것이 적절하다.
- 슬롯 생성 범위는 앞으로 4주치가 적절하다.
- 반복 규칙 권한은 센터 매니저 전용으로 두는 것이 적절하다.
- 트레이너는 본인에게 배정된 회차에 한해 제한적인 예외 처리만 허용하는 것이 적절하다.
- 1차 범위에서 트레이너 권한은 보수적으로 시작하고, 운영 피드백이 쌓이면 확대 여부를 검토한다.
