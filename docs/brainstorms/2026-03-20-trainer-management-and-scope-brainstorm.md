# Trainer Management And Scope

## What We're Building
트레이너를 별도 독립 루트 엔티티로 먼저 만들기보다, 기존 `users` 기반 계정 모델 위에서 `ROLE_TRAINER` 사용자를 관리하는 방식으로 트레이너 관리 기능을 추가한다. 운영 화면에서는 일반 사용자 관리와 분리된 별도 "트레이너 관리" 메뉴를 제공해 센터 운영자가 트레이너를 업무 단위로 다룰 수 있게 한다.

이 화면은 단순 계정 목록이 아니라 운영 현황 보드 성격을 가진다. 목록에는 트레이너 이름, 상태, 연락처, 담당 회원 수, 오늘 예약 수를 보여준다. 여기서 담당 회원 수는 회원권 수가 아니라 중복 제거된 회원 수 기준이며, 오늘 예약 수는 해당 트레이너에게 할당된 회원들의 오늘 예약 건수 기준으로 본다.

## Why This Approach
처음부터 별도 `Trainer` 도메인을 독립적으로 도입하면 `userId`와 `trainerId`를 이중 관리하게 되고, 현재 존재하는 `assignedTrainerId`, `ROLE_TRAINER`, 트레이너 범위 예약 조회 구조와 충돌하거나 중복이 생긴다. 지금 필요한 요구는 트레이너 계정 생성/관리, 담당 회원 연결, 담당 회원 예약 조회이므로 `User as Trainer` 모델이 더 단순하고 현실적이다.

화면은 별도 메뉴로 분리하는 것이 맞다. 요구사항 문서상 프론트데스크는 회원 응대와 현장 운영이 중심이고, 트레이너 계정/권한 자체를 전면 관리하는 주체는 아니다. 따라서 UI는 분리하되, 권한은 보수적으로 설계한다.

## Key Decisions
- 트레이너의 정체성은 별도 독립 엔티티보다 `users + ROLE_TRAINER`로 본다: 인증 주체와 업무 주체를 같은 `userId`로 유지해 현재 모델과 정합성을 맞춘다.
- 트레이너 관리는 별도 메뉴로 분리한다: 일반 사용자 관리와 다른 운영 맥락을 반영한다.
- `SUPER_ADMIN`은 전체 센터의 트레이너 CRUD를 가진다: 센터에서 해결 불가능한 계정/운영 문제를 상위에서 복구할 수 있어야 한다.
- `CENTER_ADMIN`은 자기 센터의 트레이너 CRUD를 가진다: 실질적인 센터 운영 책임 주체로 본다.
- `DESK`는 트레이너 관리 메뉴를 조회 전용으로 사용한다: 운영상 필요한 가시성은 제공하되 계정/권한 변경 권한은 주지 않는다.
- `TRAINER`는 트레이너 관리 메뉴에 접근하지 않는다: 본인 스케줄과 본인 담당 회원 예약 처리에 집중한다.
- 담당 회원 연결은 기존 `member_memberships.assignedTrainerId`를 유지한다: 새로운 연결 축을 만들지 않는다.
- 트레이너가 보는 예약 범위는 본인에게 할당된 회원 기준으로 제한한다: 현재 예약/회원 범위 제어 방향을 그대로 확장한다.
- 트레이너 관리 목록의 담당 회원 수는 중복 제거된 회원 수 기준으로 집계한다: 운영 의미가 더 자연스럽다.
- 트레이너 관리 목록의 오늘 예약 수는 본인 담당 회원들의 오늘 예약 건수 기준으로 집계한다: 현장 운영 관점의 워크로드 파악에 더 적합하다.
- 트레이너 관리 목록의 오늘 예약 수는 오늘 날짜의 `CONFIRMED` 예약 건수만 포함한다: 취소/완료/노쇼를 제외한 현재 유효 예약만 보여주기 위함이다.
- 연락처는 우선 계정 속성으로 취급한다: 현재 `users`에 없으면 `users.phone` 추가를 우선 검토하고, 별도 프로필 테이블은 추후 필요 시 도입한다.
- 장기적으로 `trainer_schedules`는 문자열 `trainerName`보다 식별자 기반 참조로 정리한다: 트레이너 계정과 스케줄을 안정적으로 연결하기 위함이다.

## Open Questions
- 트레이너 연락처가 현재 `users` 테이블에 이미 존재하는지 확인이 필요하다.

## Resolved Questions
- 오늘 예약 수는 오늘 날짜의 `CONFIRMED` 예약 건수만 포함한다.
- 트레이너 연락처는 별도 트레이너 전용 저장소가 아니라 `users.phone`으로 관리한다.
- 데스크는 트레이너 관리 화면을 조회 전용으로 사용하며, 상세 화면에서도 로그인 ID 같은 계정성 정보는 노출하지 않는다.
