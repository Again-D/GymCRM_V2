# Membership Reservation Member Selection Flow Brainstorm

## What We're Building
회원권 업무 탭과 예약 관리 탭의 진입 흐름을 리팩토링한다. 현재는 두 탭이 선택된 회원을 전제로 동작해서, 사용자가 해당 업무를 하려면 회원관리 탭으로 이동해 회원을 선택한 뒤 다시 돌아와야 한다.

리팩토링 목표는 이 불필요한 왕복 이동을 제거하는 것이다. 회원관리 탭에서 회원을 선택해 바로 회원권 업무나 예약 관리로 이동하는 빠른 경로는 유지하되, 사용자가 사이드바에서 직접 해당 탭으로 들어왔을 때도 탭 내부에서 회원을 검색하고 선택해 업무를 시작할 수 있게 한다.

## Why This Approach
핵심 문제는 기능 발견성이 아니라 업무 흐름의 단절이다. 회원권 업무와 예약 관리를 회원관리 탭에 종속시키거나 숨기면 구조는 단순해질 수 있지만, 실제 운영 흐름은 계속 우회하게 된다.

사이드바에서 두 탭을 독립 기능으로 유지하면서, 미선택 상태에서는 탭 내부에서 회원을 고르게 하면 정보 구조를 흔들지 않고 왕복 이동도 제거할 수 있다. 이 방식은 현재 회원관리 탭의 빠른 액션 패턴과도 충돌하지 않는다.

## Approaches Considered
### Approach A: 회원관리 하위 토글 메뉴
- 사이드바에서 회원관리를 선택했을 때만 회원권 업무/예약 관리 메뉴를 하위 목록처럼 노출.
- 장점: 메뉴 구조가 더 응집적으로 보일 수 있음.
- 단점: 실제로는 여전히 회원관리 진입과 회원 선택을 강제하게 되어 왕복 이동 문제를 충분히 줄이지 못함.

### Approach B: 사이드바에서 숨기고 회원관리에서만 진입
- 회원권 업무/예약 관리를 독립 탭에서 제거하고 회원관리 탭 액션으로만 진입.
- 장점: 구조는 가장 단순함.
- 단점: 기능 발견성이 낮아지고, 두 업무가 회원관리의 부속처럼 과도하게 종속됨.

### Approach C (Selected): 독립 탭 유지 + 탭 내부 회원 선택
- 사이드바에서 회원권 업무/예약 관리를 계속 노출.
- 회원이 이미 선택돼 있으면 해당 회원 컨텍스트로 바로 진입.
- 회원이 선택되지 않았으면 각 탭 상단에서 간단 검색과 단일 선택 리스트로 회원을 고른 뒤 업무를 시작.
- 장점: 왕복 이동을 제거하면서 현재 빠른 이동 경로도 유지.
- 단점: 각 탭에 진입용 회원 선택 UI가 추가됨.

## Key Decisions
- `회원권 업무`, `예약 관리`는 사이드바에서 계속 독립 탭으로 노출한다.
- 두 탭은 `selectedMember`가 있으면 즉시 업무 화면을 보여준다.
- `selectedMember`가 없으면 탭 내부에서 회원 검색/선택 UI를 먼저 보여준다.
- 탭 내부 회원 선택 UI는 `간단 검색 + 단일 선택 리스트` 수준으로 제한한다.
- 회원관리 탭의 액션 버튼을 통한 진입 흐름은 그대로 유지한다.
- 예약 관리 탭도 검색 대상을 미리 `회원권 보유 회원`으로 제한하지 않는다.
- 예약 가능 여부는 검색 단계에서 숨기지 않고, 회원 선택 후 상태 메시지로 설명한다.

## UX Guardrails
- 회원관리 탭은 전체 탐색/편집 중심 화면으로 남긴다.
- 회원권 업무와 예약 관리는 `대상 회원 선택 후 후속 업무 처리`에 집중한다.
- 회원 선택 UI는 회원관리 탭 전체 기능을 복제하지 않는다.
- 현재 선택 회원이 있는 경우 상단 요약과 `변경` 액션만 제공해 업무 집중도를 유지한다.
- 미선택 상태에서도 사용자가 “여기서 바로 시작할 수 있다”는 점이 즉시 이해되어야 한다.

## Reservation Search Policy
- 예약 관리 탭의 회원 검색은 전체 회원 기준으로 동작한다.
- 검색 결과에서 회원을 숨기지 않는다.
- 예약 가능 회원권이 없거나 예약 생성이 불가능한 경우, 선택 이후 화면에서 이유를 명확히 노출한다.
- 필요 시 향후 `예약 가능 회원만 보기` 같은 보조 필터를 추가할 수 있지만, 초기 범위에는 포함하지 않는다.

## Risks We Are Intentionally Avoiding
- 검색 단계에서 일부 회원을 숨겨 “왜 검색되지 않는지” 알 수 없게 되는 UX 혼란.
- UI 검색 조건과 예약 생성 검증 조건이 분리되어 정책이 어긋나는 문제.
- 회원권 업무/예약 관리 탭이 회원관리 화면을 중복 복제해 책임이 흐려지는 문제.
- 선택된 회원이 없을 때 기능 자체가 보이지 않거나 갑자기 나타나는 불안정한 네비게이션.

## Success Criteria
- 사용자가 회원관리 탭으로 먼저 이동하지 않아도 회원권 업무/예약 관리를 시작할 수 있다.
- 회원관리 탭에서 회원을 선택하고 넘어오는 빠른 흐름은 그대로 유지된다.
- 회원권 업무/예약 관리 탭에서 회원 선택 UI가 과도하게 무겁지 않고, 대상 회원 하나를 빠르게 고를 수 있다.
- 예약 관리 탭에서 검색 누락으로 오해할 만한 상황이 줄어든다.
- 각 탭의 역할이 명확하게 유지되고, 회원관리 기능 중복이 최소화된다.
