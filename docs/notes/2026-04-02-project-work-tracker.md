## 프로젝트 개요

GymCRM_V2는 체육관 운영 관리 시스템(CRM)으로, 회원/상품/회원권/예약/출석/정산/CRM/트레이너 관리 등 핵심 데스크 업무를 B2B 관리자 포털로 제공한다. Spring Boot(백엔드) + React/TypeScript(프론트엔드) + PostgreSQL + Redis 스택으로 구성되어 있다.

### Week 10: 2026-04-02 (현재) — 회원권 홀딩 기능 개선
> 현재 계획 및 초기 진행 중 (`docs/plans/2026-04-01-004-feat-membership-holding-improvements-plan.md`)

**목표:**
1. **프론트엔드 홀딩 예외처리 (Soft Override):** 기존의 하드 블록(최대 일수/횟수 제한)을 경고창(Soft Warning)으로 완화하고, 승인 시 초과 홀딩을 허용하는 UI 적용 (`overrideLimits` 플래그).
2. **백엔드 검증 로직 완화:** `overrideLimits` 파라미터가 유효할 경우, 제약 조건을 무시하고 홀딩을 승인하도록 `MembershipHoldService` 수정.
3. **홀딩 자동 해제 및 SMS 알림 스케줄러:** 매일 00:05 AM에 만료된 홀딩건을 즉시 찾아내 `RESUMED` 상태로 자동 변경하고 회원에게 SMS 통지를 전송하는 배치 처리(`@Scheduled`) 도입.
